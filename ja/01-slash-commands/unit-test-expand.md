# ユニットテストを拡張

プロジェクトのテスティングフレームワークに適応させて、既存のユニットテストを拡張します：

1. **カバレッジを分析**: カバレッジレポートを実行して、テストされていないブランチ、エッジケース、低カバレッジエリアを特定
2. **ギャップを特定**: 論理ブランチ、エラーパス、境界条件、null/空入力についてコードを確認
3. **テストを記述** (プロジェクトのフレームワークを使用)：
   - Jest/Vitest/Mocha (JavaScript/TypeScript)
   - pytest/unittest (Python)
   - Go testing/testify (Go)
   - Rust test framework (Rust)
4. **特定シナリオをターゲット**：
   - エラーハンドリングと例外
   - 境界値 (最小/最大、空、null)
   - エッジケースとコーナーケース
   - 状態遷移と副作用
5. **改善を検証**: カバレッジを再実行し、測定可能な増加を確認

新しいテストコードブロックのみを提示します。既存のテストパターンと命名規則に従ってください。
