# コード レビュー チェックリスト

## セキュリティ チェックリスト
- [ ] ハードコードされた認証情報またはシークレットなし
- [ ] すべてのユーザー入力に対する入力検証
- [ ] SQL インジェクション防止（パラメーター化されたクエリ）
- [ ] 状態を変更する操作でのCSRF保護
- [ ] 適切なエスケープでのXSS防止
- [ ] 保護されたエンドポイントでの認証チェック
- [ ] リソースでの認可チェック
- [ ] セキュアなパスワード ハッシング（bcrypt、argon2）
- [ ] ログに機密データなし
- [ ] 強制HTTPS

## パフォーマンス チェックリスト
- [ ] N+1クエリなし
- [ ] インデックスの適切な使用
- [ ] キャッシング有益な場所に実装
- [ ] メイン スレッドでのブロッキング操作なし
- [ ] Async/awaitは正しく使用
- [ ] 大規模なデータセットはページネーション
- [ ] データベース接続プール
- [ ] 正規表現が最適化
- [ ] 不要なオブジェクト生成なし
- [ ] メモリ リークの防止

## 品質チェックリスト
- [ ] 関数 < 50行
- [ ] 明確な変数の命名
- [ ] コード重複なし
- [ ] 適切なエラー処理
- [ ] コメントはWHYを説明します（WHATではなく）
- [ ] 本番環境ではconsole.logsなし
- [ ] タイプ チェック（TypeScript/JSDoc）
- [ ] SOLID原則に従う
- [ ] デザイン パターンが正しく適用される
- [ ] 自己説明コード

## テスト チェックリスト
- [ ] ユニット テストが作成される
- [ ] エッジ ケースがカバーされている
- [ ] エラー シナリオがテスト
- [ ] 統合テストが存在
- [ ] カバレッジ > 80%
- [ ] 不安定なテストなし
- [ ] モック外部依存関係
- [ ] 明確なテスト名
