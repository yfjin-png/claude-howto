# チェックポイント例

Claude Codeでチェックポイントを効果的に使用する実世界の例。

注: チェックポイントはすべてのユーザープロンプトで自動的に作成されます。手動で保存する必要はありません。巻き戻すには、`Esc`を2回押す（`Esc+Esc`）か`/rewind`を使用してチェックポイントブラウザを開きます。

## 例1: データベース移行

### シナリオ
MySQLからPostgreSQLへの移行が必要ですが、直接移行と段階的なデュアルライトアプローチの両方を探索したいです。

### ワークフロー

```
ユーザー: MySQLからPostgreSQLへの直接移行を実装しましょう
      [チェックポイントA自動作成]

Claude: 直接移行を実装します...
[データベースドライバの変更、クエリの更新、スキーマの変更]

ユーザー: テストを実行
      [チェックポイントB自動作成]

Claude: テストを実行しています...
❌ 15のテスト失敗 - コネクションプール問題

ユーザー: このアプローチに問題があります。代わりにデュアルライトパターンを試してみてください。
[ユーザーがEsc+Escを押し、チェックポイントAを選択、「コードと会話を復元」を選択]

ユーザー: ここで両方のデータベースに書き込むデュアルライトパターンを実装します

Claude: デュアルライト移行を実装します...
[デュアルライトレイヤー、段階的な移行戦略を実装]

ユーザー: テストを実行

Claude: テストを実行しています...
✅ すべてのテストが成功

ユーザー: デュアルライトアプローチが明らかに安全です。これをコミットしましょう。
```

## 例8: チェックポイントから要約を使用

### シナリオ
長いデバッグセッションの後、会話を圧縮したいけれど、コンテキストを保存します。

### ワークフロー

```
ユーザー: [20以上のメッセージのデバッグと探索の後]

[ユーザーがEsc+Escを押し、早期チェックポイントを選択、「ここから要約」を選択]
[オプションで命令を提供：「何を試したか、そして何が機能したかに焦点を当てる」]

Claude: [そのポイント以降の会話の要約を生成]
[元のメッセージはトランスクリプトに保持]
[要約は見える会話を置き換え、コンテキストウィンドウ使用を削減]

ユーザー: 今、機能したアプローチで続行しましょう。
```

## キーテイクアウェイ

1. **チェックポイントは自動**: すべてのユーザープロンプトがチェックポイント作成 -- 手動保存は不要
2. **Esc+Escまたは/rewindを使用**: チェックポイントブラウザにアクセスするこれら2つの方法
3. **正しい復元オプションを選択**: 必要に応じてコード、会話、両方、または要約を復元
4. **実験を恐れない**: チェックポイントは根本的な変更を試すのを安全にします
5. **Gitと組み合わせる**: 探索にチェックポイント、最終化された作業にgit
6. **長いセッションを要約**: 「ここから要約」を使用して会話を管理可能に保つ
