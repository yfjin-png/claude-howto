## 概要

Claude Codeの高度な機能は、プランニング、推論、自動化、制御メカニズムで基本機能を拡張します。これらの機能は複雑な開発タスク、コードレビュー、自動化、マルチセッション管理のための高度なワークフローを可能にします。

**主要な高度な機能:**
- **プランニングモード**: コーディング前に詳細な実装計画を作成
- **拡張思考**: 複雑な問題に対して深い推論
- **オートモード**: バックグラウンド安全分類器が実行前に各アクションをレビュー（研究プレビュー）
- **バックグラウンドタスク**: 会話をブロックすることなく長い操作を実行
- **許可モード**: Claudeが何をできるかを制御（`default`、`acceptEdits`、`plan`、`auto`、`dontAsk`、`bypassPermissions`）
- **プリントモード**: 自動化とCI/CDのための非インタラクティブ実行（`claude -p`）
- **セッション管理**: 複数の作業セッションを管理
- **インタラクティブ機能**: キーボードショートカット、複数行入力、コマンド履歴
- **音声入力**: 20言語STT対応のプッシュトークボイス入力
- **チャネル**: MCPサーバーは実行中のセッションにメッセージをプッシュ（研究プレビュー）
- **リモートコントロール**: Claude.aiまたはClaudeアプリからClaude Codeを制御
- **ウェブセッション**: claude.ai/codeでブラウザでClaude Codeを実行
- **デスクトップアプリ**: ビジュアルdiffレビューと複数セッション用のスタンドアロンアプリ
- **タスクリスト**: コンテキスト圧縮間で永続的なタスク追跡
- **プロンプト提案**: コンテキストに基づくスマートコマンド提案
- **Gitワークツリー**: 並列作業のための隔離されたワークツリーブランチ
- **サンドボックス**: OS レベルのファイルシステムとネットワーク隔離
- **管理設定**: plist、Registry、または管理ファイル経由のエンタープライズ配置
- **設定**: JSON設定ファイルで動作をカスタマイズ

## プランニングモード

プランニングモードを使用すれば、Claudeは複雑なタスクを実装する前に考える、詳細な計画が作成します。

### プランニングモードとは何ですか？

プランニングモードは2段階のアプローチです：
1. **プランニング段階**: Claudeはタスクを分析し、詳細な実装計画を作成
2. **実装段階**: 承認後、Claudeが計画を実行

### プランニングモードを使用するべき時

✅ プランニングモードを使用:
- 複雑なマルチファイルリファクタリング
- 新しい機能実装
- アーキテクチャ変更
- データベース移行
- 主要なAPI再設計

❌ プランニングモードを使用しないでください:
- 単純なバグ修正
- フォーマット変更
- シングルファイル編集
- クイッククエリ

### プランニングモードを有効化

**スラッシュコマンド**:
```bash
/plan ユーザー認証システムを実装
```

**CLIフラグ**:
```bash
claude --permission-mode plan
```

**デフォルトとして設定**:
```json
{
  "permissions": {
    "defaultMode": "plan"
  }
}
```

### プランニングモードの利点

- **構造化された実装アプローチ**: 実装ステップの詳細な分解
- **レビューと承認**: 実行前に計画を承認または調整
- **リスク特定**: 実装前に潜在的な問題を特定
- **明確なフェーズ**: 整理された実装フェーズとマイルストーン

## 拡張思考

拡張思考を使用すれば、Claudeは複雑な問題について解決策を提供する前により多くの時間を推論に費やすことができます。

### 拡張思考とは何ですか？

拡張思考は、Claudeが以下を行う意図的なステップバイステップの推論プロセスです：
- 複雑な問題を分解
- 複数のアプローチを検討
- トレードオフを評価
- エッジケースを通じて推論

### 拡張思考を有効化

**キーボードショートカット**:
- `Option + T` (macOS) / `Alt + T` (Windows/Linux) - 拡張思考を切り替え

**自動有効化**:
- すべてのモデルに対してデフォルトで有効（Opus 4.6、Sonnet 4.6、Haiku 4.5）
- Opus 4.6: `low` (○)、`medium` (◐)、`high` (●)、`max` (Opus 4.6のみ)の努力レベルでの適応推論
- その他のモデル: 最大31,999トークンの固定予算

## オートモード

オートモードは研究プレビュー許可モード（2026年3月）で、バックグラウンド安全分類器を使用して実行前に各アクションをレビューします。Claudeが危険な操作を自律的に作業しながらブロックできます。

### 要件

- **プラン**: チーム、エンタープライズ、またはAPI（ProやMaxプランでは利用不可）
- **モデル**: Claude Sonnet 4.6またはOpus 4.6
- **プロバイダー**: Anthropic APIのみ（BedRock、Vertex、またはFoundryではサポートされていません）
- **分類器**: Claude Sonnet 4.6で実行（追加トークンコスト）

---

## バックグラウンドタスク

バックグラウンドタスクを使用すれば、長時間実行される操作は会話をブロックするなく実行できます。

### バックグラウンドタスクとは何ですか？

バックグラウンドタスクは非同期的に実行されます：
- 長いテストスイート
- ビルドプロセス
- データベース移行
- デプロイメントスクリプト
- 分析ツール

---

## スケジュール済みタスク

スケジュール済みタスクを使用すれば、定期的なスケジュール上または特定の時間にプロンプトを自動的に実行できます。タスクはセッション対象 — Claudeコードがアクティブな場合に実行され、セッションが終了すると消去されます。v2.1.72+から利用可能。

### `/loop`コマンド

```bash
# 明示的な間隔
/loop 5m デプロイが終わったかをチェック

# 自然言語
/loop 30分ごとにビルドステータスをチェック
```

---

## 許可モード

許可モードは、明示的な承認なくClaudeが実行できるアクションを制御します。

### 利用可能な許可モード

| モード | 動作 |
|--------|------|
| `default` | ファイルのみを読む、他のすべてのアクションについてプロンプト |
| `acceptEdits` | ファイルを読んで編集、コマンドについてプロンプト |
| `plan` | ファイルのみを読む（リサーチモード、編集なし） |
| `auto` | バックグラウンド安全分類器チェック付きのすべてのアクション（研究プレビュー） |
| `bypassPermissions` | すべてのアクション、許可チェックなし（危険） |
| `dontAsk` | 事前承認ツールのみを実行、その他はすべて拒否 |

## ヘッドレスモード

プリントモード（`claude -p`）を使用すれば、Claude Codeは自動化とCI/CDに完璧です。

### プリントモードとは何ですか？

プリントモードは以下を有効にします：
- 自動スクリプト実行
- CI/CD統合
- バッチ処理
- スケジュール済みタスク

## セッション管理

複数のClaude Codeセッションを効果的に管理できます。

### セッション管理コマンド

| コマンド | 説明 |
|---------|------|
| `/resume` | IDまたは名前で会話を再開 |
| `/rename` | 現在のセッション命名 |
| `/fork` | 現在のセッションを新しいブランチに分岐 |
| `claude -c` | 最新の会話を続行 |
| `claude -r "session"` | 名前またはIDでセッション再開 |

## インタラクティブ機能

Claude Codeは効率のためのキーボードショートカットをサポートします。

### キーボードショートカット

| ショートカット | 説明 |
|---------|------|
| `Ctrl+C` | 現在の入力/生成をキャンセル |
| `Ctrl+D` | Claude Codeを終了 |
| `Ctrl+G` | 外部エディタで計画を編集 |
| `Ctrl+L` | ターミナル画面をクリア |
| `Ctrl+O` | 詳細出力を切り替え（推論を表示） |
| `Ctrl+R` | リバース検索履歴 |
| `Ctrl+T` | タスクリストビューを切り替え |
| `Ctrl+B` | バックグラウンド実行タスク |
| `Esc+Esc` | コード/会話を巻き戻す |
| `Shift+Tab` / `Alt+M` | 許可モードを切り替え |
| `Option+P` / `Alt+P` | モデルを切り替え |
| `Option+T` / `Alt+T` | 拡張思考を切り替え |

## 音声入力

音声入力は、Claude Codeにプッシュトークボイス入力を提供します。

### 音声入力を有効化

```
/voice
```

### 機能

| 機能 | 説明 |
|------|------|
| **プッシュトーク** | キーを押して録音、リリースして送信 |
| **20言語** | 音声認識が20言語をサポート |
| **カスタムキーバインディング** | `/keybindings`でプッシュトークキーを設定 |
| **アカウント要件** | STT処理用Claude.aiアカウントが必要 |

## チャネル

チャネルは研究プレビュー機能で、MCPサーバー経由で実行中のセッションに外部サービスからのイベントをプッシュします。

### チャネルを購読

```bash
# 起動時にチャネルプラグインを購読
claude --channels discord,telegram

# 複数のソースを購読
claude --channels discord,telegram,imessage,webhooks
```

### サポートされた統合

| 統合 | 説明 |
|-----|------|
| **Discord** | セッションでDiscordメッセージを受信および応答 |
| **Telegram** | セッションでTelegramメッセージを受信および応答 |
| **iMessage** | セッションでiMessage通知を受信 |
| **Webhooks** | 任意のwebhookソースからイベントを受信 |

## Chrome統合

Chrome統合はClaude Codeをブラウザに接続して、ライブウェブ自動化とデバッグを行います。

### Chrome統合を有効化

```bash
claude --chrome      # Chrome接続を有効化
claude --no-chrome   # Chrome接続を無効化
```

### 機能

| 機能 | 説明 |
|------|------|
| **ライブデバッグ** | コンソールログを読む、DOMを検査、JavaScript をリアルタイムでデバッグ |
| **設計検証** | レンダリングされたページを設計モックアップと比較 |
| **フォーム検証** | フォーム送信、入力検証、エラー処理をテスト |
| **ウェブアプリテスト** | 認証されたアプリ（Gmail、Google Docs、Notionなど）と相互作用 |
| **データ抽出** | ウェブページからコンテンツをスクレイプおよび処理 |
| **セッション記録** | ブラウザインタラクションをGIFファイルとして記録 |

## リモートコントロール

リモートコントロールを使用すれば、電話、タブレット、またはいかなるブラウザからローカル実行中のセッションを続行できます。

### リモートコントロールを開始

```bash
# デフォルトセッション名で開始
claude remote-control

# カスタム名で開始
claude remote-control --name "Auth Refactor"
```

### セキュリティ

- **インバウンドポートなし** マシンで開きます
- **TLS経由の送信HTTPSのみ**
- **スコープ付きクレデンシャル** 複数の短命、狭くスコープ付きトークン
- **セッション隔離** 各リモートセッションは独立

claude --remote "新しいAPIエンドポイントを実装"

## デスクトップアプリ

Claude Codeデスクトップアプリは、ビジュアルdiffレビュー、並列セッション、統合コネクタを備えた スタンドアロンアプリを提供します。

### コア機能

| 機能 | 説明 |
|------|------|
| **Diffビュー** | インラインコメント付きファイルごとのビジュアルレビュー；Claudeはコメントを読んで修正 |
| **アプリプレビュー** | 自動的に開発サーバーを起動してライブ検証用の埋め込みブラウザ |
| **PRモニタリング** | GitHub CLI統合でCI失敗を自動修正、チェックが成功すると自動マージ |
| **並列セッション** | サイドバーに複数セッション、自動Git ワークツリー隔離 |
| **スケジュール済みタスク** | 定期的なタスク（時間単位、日単位、平日、週単位）がアプリを開いている間実行 |
| **豊富なレンダリング** | コード、markdown、図レンダリング、構文強調表示付き |

## プロンプト提案

プロンプト提案は、git履歴と現在の会話コンテキストに基づいてグレーアウトされた例コマンドを表示します。

### その動作方法

- 提案は入力プロンプト下にグレーアウトされたテキストとして表示
- 提案を受け入れるには`Tab`を押す
- `Enter`を押して受け入れと即座に送信

### プロンプト提案を無効化

```bash
export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

Gitワークツリーを使用すれば、隔離されたワークツリーでClaude Codeを開始でき、スタッシュまたはスイッチなしで並列作業が可能です。

## ベストプラクティス

### プランニングモード
- ✅ 複雑なマルチステップタスク用に使用
- ✅ 承認前に計画を確認
- ✅ 必要に応じて計画を変更
- ❌ 単純なタスク用に使用しないでください

### 拡張思考
- ✅ 建築決定用に使用
- ✅ 複雑な問題解決用に使用
- ✅ 思考プロセスを確認
- ❌ 単純なクエリ用に使用しないでください

### バックグラウンドタスク
- ✅ 長時間実行される操作用に使用
- ✅ タスク進捗を監視
- ✅ タスク失敗を適切に処理
- ❌ 同時タスクを開始しすぎないでください

### 許可
- ✅ コードレビュー用に`plan`を使用（読み取り専用）
- ✅ インタラクティブ開発用に`default`を使用
- ✅ 自動化ワークフロー用に`acceptEdits`を使用
- ✅ 自動作業用に`auto`を使用しましょう
- ❌ `bypassPermissions`を絶対に必要でない限り使用しないでください

### セッション
- ✅ 異なるタスク用に別々のセッションを使用
- ✅ 重要なセッション状態を保存
- ✅ 古いセッションをクリーンアップ
- ❌ 関連のない作業を1つのセッションで混ぜないでください
